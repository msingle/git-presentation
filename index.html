So, What Is Git?

At it's core, it's a distributed revision control system

As well as a source code management system

Distributed?

No central service to manage the revisions

This means that every person has a full copy of the repository

It can be pretty awesome, but it's also got some cons

  • PRO: can work offline!
  • PRO: don't need to contact a central server to commit changes or view history
  • PRO: every copy is now a backup of the code
  • CON: repos can get HUGE

Getting Started

Cloning Your Way To Victory

AKA: The Star Wars Approach
Not Really

Cloning Gets You Started

git clone <repository url>

Now you've got a copy of the code!

Examples Repo URLs

  • /home/sean/some-project
  • artifact-server:porcupine.git
  • ssh://[user@]host:path/to/repo.git

GitHub

GitHub tends to use the following format:

git@github.com:<your github username>/<repo name>.git

This presentation can be cloned from:

git clone git@github.com:seanhagen/git-presentation.git

Tips!

Here are a few flags that you might find useful when cloning:

  • --branch <name>
    clone a specific branch
  • --depth <num>
    only clone the specified number of revisions

depth: useful when you want to get a shallow copy, like if you're deploying a demo to server

NOW WHAT?

What's The Status?

git status

Gives you a list of the following:

  • Changes that git are going to be comittted ( more later )
  • Files git isn't tracking yet

Adding Files

This lets git know what to keep track of

git add path/to/file

Removing Files

Tell git you don't want it to track it anymore, and to remove it from the working tree

git rm path/to/file
